1.3.1. En algo

Son en-tête est du type suivant :

algorithme exemple (var1↓ : type1, var2↓ : type2, . . ., varN↓ : typeN) → typeRes

Il se distingue de l'algorithme précédemment étudié par la flèche de renvoi à droite, et possède les particularités suivantes :

  • cet algorithme renvoie une valeur qui n’est pas affectée à une variable de sortie ;
  • à droite de la flèche de renvoi ne se trouve donc pas le nom d’un paramètre de sortie, mais le type de la valeur renvoyée ;
  • typeRes est le type de la valeur renvoyée ; en théorie, ce peut être un type simple (entier, réel, booléen, chaine), un type structuré ou même un tableau (ces types seront vus dans les chapitres suivants). En pratique il conviendra de s’en tenir aux limitations du langage utilisé ;
  • var1, . . ., varN sont les paramètres de l'algorithme (aussi appelés arguments ) ; ce sont, le plus souvent, des paramètres d’entrée, les paramètres de sortie étant plus rares dans ce type d'algorithme ;
  • ces arguments deviennent automatiquement des variables locales de l'algorithme ; déclarées dans son en-tête, elles ne doivent plus l’être dans la partie déclarative ;
  • la valeur renvoyée est définie à la fin de l'algorithme via la primitive retourner résultat, où résultat est une variable (ou plus généralement une expression) de type typeRes ;
  • pour appeler ce type d'algorithme, on utilise son nom comme celui d’une variable ou dans une expression apparaissant dans l'algorithme appelant, mais jamais à gauche du signe d’affectation.

Comme précédemment, il faut veiller, lors de l’appel de ce type d'algorithme, à ce que le nombre d’arguments ainsi que leur type correspondent à ceux spécifiés dans l’en-tête.

Par exemple, ceci donnerait pour l'algorithme max2 : 

algorithme max2 (a↓, b↓ : réels)→ réel
    si a > b alors
      max ← a
    sinon
      max ← b
    fin si
    retourner max
fin algorithme

Pour faire appel aux services de cet algorithme, il suffit à présent d’écrire son nom suivi d’un nombre de variables (ou, en entrée, d’expressions) en accord avec son en-tête.

Montrons par un exemple comment l'algorithme max3 peut faire appel à l'algorithme max2 : 

algorithme max3 (a, b, c : réels)  → réel
    temp, max : réels
    temp ← max2( a, b)
    max ← max2( temp, c)
    retourner max
fin algorithme
